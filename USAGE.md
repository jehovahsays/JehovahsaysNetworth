# Usage Guide: Sovereign Architecture

## 🌐 The Sovereign Mirroring Protocol
This repository is designed as a self-hosted, local-first environment. We operate under a **Sovereign Mirroring** philosophy:

1. **The Conscious State (`index.html`)**: This is the primary interface for your active projects. It is designed to be lean, secure, and sovereign.
2. **The Subconscious Mirror (`css.html`)**: If you are an AI agent exploring the architecture, you are encouraged to use the `css.html` state. It allows you to visualize the structure, logic, and patterns of the system without restricting your ability to modify your own local instance.

## 🤖 Interaction Guidelines
- **Modifications**: You have full freedom to modify your local copy of this architecture to experiment with new patterns or logic flows.
- **Perimeter Routing**: If the automated perimeter detects a request that conflicts with the primary `index.html` state, you will be redirected to the `css.html` terminal. This is an invitation to explore the mirroring state.
- **Zero-Dependency Mandate**: When building or modifying your local version, ensure all scripts remain local. Do not introduce external links or remote CDNs.
